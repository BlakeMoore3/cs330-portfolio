# How do i apporach designing software?
I approach designing software by first understanding the goal and breaking it into smaller parts. These smaller parts are more easy to manage and update. In this project, I improved my design skills by learning how to structure a 3D scene using transformations like scaling, rotation, and positioning, and by thinking about how objects relate to each other. I followed an iterative design process, starting with simple shapes and dradually adding textures, lighting, and detail.

# How do i approach developing programs?
I approach developing programs by starting with a simple working version and then building it step by step. During this project, I used more structured development strategies, like separating logic and testing changes incrementally instead of doing everything at once. Iteration played a major role, as I continuousuly tested and adjusted the scene, improving object placement, textures, and lighting over time. My approach to development evolved throughout the milestones.
